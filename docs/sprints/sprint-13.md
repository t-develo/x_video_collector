# Sprint 13: Responsive & Polish

## 目的

レスポンシブ対応と UI のポリッシュ。

## ブランチ

`feature/sprint13/responsive-polish`

## タスク

### 13.1 デスクトップファーストのレスポンシブ対応

- ブレークポイント定義（CSS カスタムプロパティ or メディアクエリ）
  - Desktop: >= 1200px（基準）
  - Tablet: 768px - 1199px
  - Mobile: < 768px
- ヘッダー / ナビゲーションのレスポンシブ化
- 動画カードグリッドのカラム数調整
- フォームレイアウトの調整

### 13.2 アクセシビリティ

- キーボードナビゲーション
- ARIA ラベル
- フォーカス管理
- 色コントラスト確認

### 13.3 パフォーマンス

- 画像の遅延読み込み（Intersection Observer）
- 不要な再レンダリング防止
- CSS アニメーションの最適化

### 13.4 エラーハンドリング強化

- ネットワークエラー時の UI 表示
- API エラーの統一的なハンドリング
- リトライ UI（手動リトライボタン）

### 13.5 UI ポリッシュ

- ローディングスケルトン
- トランジション・アニメーション
- アイコン（SVG インライン）
- Empty state の改善

### 13.6 テスト

- レスポンシブレイアウトの確認（手動）
- アクセシビリティテスト

## 完了条件

- [ ] デスクトップ・タブレット・モバイルで適切に表示される
- [ ] キーボードで操作できる
- [ ] ローディング状態が適切に表示される
- [ ] エラー時のフィードバックが適切
- [ ] テストがすべてパスする
